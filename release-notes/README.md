# 版本说明（Release notes）

本目录存放**按版本拆分**的发行说明 Markdown，用于：

- 在仓库内集中记录各版本的**功能、修复与升级步骤**
- 发布 [Gitee 发行版](https://gitee.com/numen06/nginx-webui/releases) 时，将对应版本的 `.md` 正文复制到「发行版说明」中，与线上展示保持一致

## 与版本号对齐

应用显示的版本号以 [`backend/config.yaml`](../backend/config.yaml) 中的 `app.version` 为**唯一权威来源**。

发布新版本时的建议顺序：

1. 更新 `app.version`
2. 在本目录新增与版本号同名的文件，例如 `1.0.4.md`
3. 提交代码并打 Tag、在 Gitee 创建 Release，说明正文从该文件复制

## 文件命名约定

- 使用与 `app.version` **完全一致**的文件名：`x.y.z.md`（示例：`1.0.3.md`）
- 不使用 `v` 前缀，避免与配置中的版本字符串不一致

## 单文件内容建议

每个版本文件建议包含：版本与日期、本版亮点/新增与改进、问题修复、升级说明（Docker / 源码部署）、注意事项或破坏性变更（无则写明「无」）。
