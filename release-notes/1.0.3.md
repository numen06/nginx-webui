## 升级说明

### Docker / Docker Compose

1. 备份数据目录与证书目录（若已挂载），例如宿主机上的 `data` 与 `letsencrypt`。
2. 拉取新镜像并重建容器，例如在项目目录执行：

   ```bash
   docker compose pull
   docker compose up -d
   ```

3. 保持原有 **数据卷挂载** 不变（如 `data` → `/app/data`、`letsencrypt` → `/etc/letsencrypt`），避免丢失配置与证书。

具体端口与卷路径以当前仓库中的 [`docker-compose.yml`](../docker-compose.yml) 为准。

### 源码 / 本地运行

1. 拉取本版本代码。
2. 按仓库 [`README.md`](../README.md) 与 [`QUICKSTART.md`](../QUICKSTART.md) 安装依赖、构建前端并启动后端。
3. 若使用自定义 `backend/config.yaml`，对比仓库示例确认无新增必填项后再启动。

## 注意事项与破坏性变更

**无**。本版本未声明不兼容的配置或数据格式变更；若从较旧版本跨越多个小版本升级，仍建议先备份 `data` 与证书相关目录。

发版时应用版本号以仓库根下 [`backend/VERSION`](../backend/VERSION) 为准（与 `config.yaml` 分离，减少合并冲突）。
