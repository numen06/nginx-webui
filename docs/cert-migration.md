# 证书导出、导入与自动续签说明

多数情况下**不会**整包迁移 Certbot 目录，常见流程是：**在原环境导出证书与私钥 → 在本系统导入**。下面说明这种用法，以及若希望 **Let’s Encrypt / Certbot 自动续签**应如何处理。

## 界面向导（推荐）

在 **证书管理** 页面点击 **「迁移与自动续签」**：

1. **导出**：按提示在原环境保存证书链与私钥。  
2. **导入**：在本页面上传文件或压缩包（与「上传证书」相同接口）。  
3. **自动续签**：向导会说明导入证书与 Certbot 续签的区别；点击 **「申请证书并启用自动续签」** 时，若已导入同名记录，会先**删除该导入记录**再打开 **「申请证书」** 向导（避免域名重复），签发成功后将由本系统定时续签。

也可在第一步选择 **「无旧证书，直接申请」**，跳过导入。

---

## 一、常见做法：导出再导入

### 1. 在原服务器或面板「导出」

从原机器上取出（名称可能略有不同）：

- 完整证书链：多为 `fullchain.pem`，或合并后的 `.crt` / `.pem`
- 私钥：`privkey.pem` 或 `.key`

若原环境是 Certbot，文件一般在：

`/etc/letsencrypt/live/<域名>/fullchain.pem`  
`/etc/letsencrypt/live/<域名>/privkey.pem`

也可使用各云厂商 / 面板提供的「下载证书」功能，只要最终能拿到**链文件 + 私钥**即可。

### 2. 在本系统「导入」

进入 **证书管理 → 上传证书**：

- **文件**：分别选择证书文件与私钥；或  
- **压缩包**：将上述文件打成 zip/tar 上传，系统会尝试识别域名。

导入成功后，Nginx 即可使用 HTTPS；**到期前**可在本页 **重新上传** 替换为新文件。

---

## 二、导入后，与「自动续签 / 续期」的关系（重要）

| 情况 | 说明 |
|------|------|
| **只做导出 → 导入（PEM）** | 本机 **Certbot 里没有**这条证书的登记信息（无 lineage）。本系统的 **「续期」** 若走 Certbot，**无法**针对这条「仅导入」的证完成续签。 |
| **希望由本系统自动续签（Certbot）** | 需要让证书 **在本系统所在环境**通过 **「申请证书」** 走一遍 Let’s Encrypt（HTTP 或 DNS 验证）。签发成功后，本机会建立 Certbot lineage，**自动续期**开关与定时任务才会对这条域名生效。 |
| **暂不重新申请** | 可在证书到期前，从别处再次 **导出** 新证书，在本系统 **重新上传** 替换（与是否 Certbot 无关）。 |

**结论：**  
- **导出 / 导入** = 快速上线、不依赖迁移 Certbot 目录。  
- **要本系统里的 Certbot 自动续签** = 通常要在本机 **重新申请一次** 同域名证书（或极少数场景下由运维完整恢复 `/etc/letsencrypt`，一般客户不必做）。

---

## 三、自检（可选）

在 **证书管理** 中点击 **「测试自动续签环境」**：检测本机 `certbot` 与 `certbot renew --dry-run` 是否正常。  
通过表示**环境具备**自动续签能力；**仅导入 PEM** 的条目仍须按上文「重新申请」才会被 Certbot 接管。

---

## 四、给客户的一段话（可复制）

> 一般做法是：在原来服务器把证书和私钥 **导出**，到本系统里 **上传证书** 完成导入，网站即可启用 HTTPS。若希望 **以后由系统自动续证书**，需要在本系统对同一域名再点一次 **「申请证书」**（Let’s Encrypt），让证书在本机用 Certbot 签出来；只导入文件的话，到期前要 **重新导出再上传**，或提前在本系统申请好自动续签。

---

## 附录：何时才需要「整包 Certbot 目录」迁移？

仅当必须保留 **原 Certbot 的 renewal 配置与 lineage**、且不能在本系统重新申请时，才需要同步 `live` / `archive` / `renewal` 等目录；**日常客户支持以导出 / 导入 + 必要时重新申请为主即可。**
